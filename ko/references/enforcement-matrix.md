# 강제 수단 매트릭스

어떤 규칙을 어떤 수단으로 강제하고, 수단이 막혔을 때 어디로 강등하는지의 단일 기준표.
GitHub 설정의 실제 명령·검증 절차는 `github-setup.md` 참고.

## 규칙 × 수단

| 규칙 | 1차 수단 | private+무료 폴백 |
|---|---|---|
| squash-only 머지 | 레포 머지 설정 | 폴백 불필요 — 플랜 무관 항상 가능 |
| 기본 브랜치 직push 금지 | 브랜치 보호 | AGENTS.md 소프트 규칙 (강등 보고 필수) |
| 리뷰 1명 승인 (정규 모드) | 브랜치 보호 required reviews | AGENTS.md 소프트 + PR 템플릿 리뷰어 칸 |
| PR 제목 `타입: 설명` | AGENTS.md (에이전트가 작성) | CI 제목 검사 (정규 모드 선택지 — 빨간 X 표시, 머지 차단은 불가) |
| 검증 체크리스트 | PR 템플릿 (GitHub 자동 삽입) | 폴백 불필요 — 플랜 무관 |
| 커밋 형식 (L2만) | commit-msg 훅 | — |

## 수단별 특성

| 수단 | 플랜 제약 | 강도 | 주의점 |
|---|---|---|---|
| 레포 머지 설정 | 없음 | 하드 | `gh api -X PATCH /repos/{o}/{r}` (allow_squash_merge 등) |
| 브랜치 보호 / rulesets | public 무료 · private은 Pro/Team+ | 하드 | **private+무료에서 조용히 실패** — 적용 후 `gh api`로 재확인 필수 |
| AGENTS.md 규칙 | 없음 | 소프트 (세션마다 자동 주입) | 에이전트에게는 사실상 하드에 가깝지만, 무시 가능성은 있음 |
| PR 템플릿 | 없음 | 소프트 (가시적) | 규칙이 아니라 양식 — 규칙 본문을 복제하지 말 것 |
| CI (Actions) | private 무료 월 2,000분 | 중간 | 빨간 X까지만. 머지 차단은 브랜치 보호가 있어야 함 |
| git 훅 (core.hooksPath) | 없음 | 중간 (로컬) | clone마다 활성화 필요 — 실패가 조용함. L2에서만 사용 |

## 2겹 구조가 무너지지 않는 이유

작성 시점(AGENTS.md, 소프트)과 머지 시점(GitHub 설정, 하드)이 독립적이다. 어느 팀원의 에이전트가 규칙을 무시해도 main 오염은 머지 시점에서 막히고, 브랜치 보호가 불가한 환경에서도 squash-only는 살아남아 히스토리 형태는 유지된다.

## 강등 보고 원칙

하드 수단이 불가해 소프트로 강등되면 보고에 반드시 세 가지를 명시한다: **무엇이 강등됐나 · 왜(플랜/가시성/원격 없음/권한 없음) · 어떻게 복구하나** (public 전환 또는 플랜 업그레이드 후 스킬 재실행).
